Benford's Law — What It Tests

In naturally occurring datasets, the leading digit follows a logarithmic distribution: 1 appears 30.1% of the time, 2 appears 17.6%, down to 9 at 4.6%.

Fabricated or synthetic data deviates significantly. Chi-square < 15.5 = passes at p=0.05 (8 degrees of freedom).

P(d) = log₁₀(1 + 1/d) for d = 1, 2, ..., 9

Verdict

Devin's numbers are real telemetry.

Output tokens — the primary integrity signal — pass Benford's Law cleanly on both platforms (Devin χ²=0.52, Claude χ²=1.12). The distributions are nearly indistinguishable from the theoretical Benford curve. This is what real per-message token counts look like.

Cache read tokens — Devin fails Benford (χ²=84.00) because 69% of cache read values start with "1". This is a structural artifact, not fabrication: Devin's context windows cluster around 100K-199K tokens (values like 134,741 / 156,194 / 188,164), so the leading digit is overwhelmingly "1". Claude's context windows are more variable in size, so its cache reads spread across digits more naturally.

The ioRatio conversion (0.6783, Beta from Claude) produces pillars that match Claude's shape: lean input (3.5% of uncached), heavy cache creation (96.5%), massive cache reuse (8.1B = 14.6× uncached). These are real operator numbers.
