# Data governance

Governance defines how SignalAF collects, derives, presents, retains, and removes data. It applies user agency, data minimization, provenance, and a clear separation between live-board identity and research outputs.

These documents state the intended operating model for the observatory spine. Where implementation is not yet wired, the document says so; policy language is not evidence that a control is already active.

Research and dataset releases must use anonymized `signal-##########` identifiers and exclude codenames, display names, handles, profile URLs, and avatar URLs. The live board is the limited identity surface for opted-in operators.
